[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through how this works under the hood. 
Every user prompt first enters the Security Checkpoint. This node detects acute medical emergencies, blocks jailbreaks or prompt injections, and scrubs PII like Medicare IDs. 
If the input is safe, it goes to the Orchestrator Agent. 
The Orchestrator coordinates with two specialized sub-agents: the Medication Agent, which answers drug questions and logs vitals, and the Activity Agent, which recommends exercises.
Both sub-agents use the Model Context Protocol to fetch active prescriptions and care plans from our local MCP server.
Finally, if a senior requests something strenuous, the flow routes to a caregiver approval pause node, suspending execution until a caregiver explicitly resumes it.
